## 🎯 Mission

SNS 피로도가 높아진 사용자들이  
이미 신뢰가 형성된 소규모 관계(가족) 안에서  
진짜 소통과 생활 공유를 할 수 있는 폐쇄형 커뮤니케이션 플랫폼을 구축한다.

본 서비스는 불특정 다수를 위한 공개 SNS가 아니라,  
관계가 먼저 정의된 그룹을 전제로 한 **완벽한 프라이버시**가 보장되는 **폐쇄형 슈퍼앱**을 구축한다.

---

## 🚀 Core Features (MVP)

### 1. 가족 그룹 시스템 (Core)

- 가족 단위 그룹 생성 및 초대
- 가족 구성원 관리
- 모든 콘텐츠는 ‘가족 그룹’ 범위 안에서만 공유

---

### 2. 피드 (Private SNS)

- 가족 구성원 전체에게 기본 공개
- 글 + 이미지 기반 피드 / 댓글 / 좋아요
- 좋아요 / 댓글 기능
- 콘텐츠 상태 관리
  - 정상 / 접근 불가 / 만료

---

### 3. 블로그형 카테고리 게시판

- 특정 주제 기반 아카이빙
  - 예: `2024년 가족 여행`
- 시간 순 정렬
- 가족 단위 기록 보관용 콘텐츠

---

### 4. 공지사항

- 모든 가족 구성원 작성, 열람 가능
- 읽음 여부 상태 표시 (선택)

---

### 5. 가족 공유 캘린더

- 일정 등록 및 공유
- 구성원 태그

---

### 6. 펑(Pung) 콘텐츠

- 가볍거나 중요하지 않은 콘텐츠 전용
- 콘텐츠 수명 관리
  - 1회 조회 후 삭제
  - 3일 후 자동 만료
- 만료/접근 불가 상태에 따른 UI 분기

> 기술적 포인트:  
> 콘텐츠 수명 + 권한 + 상태 변화가 동시에 작동하는 구조 설계

## 🚫 Nice To Have

(MVP+, Scope Cutting — 설계만 고려하고 구현은 나중에)

- 가족 관계도(Interactive Family Tree)
  - 가족의 족보를 만들 수 있다. 가족 족보는 다른 족보에 포함된 가족들도 가져와서 사용할 수 있다. 즉, 가족족보는 primary한 키로 관리된다. 중요한 기능이지만 복잡할 것 같아서 일단 nice to have로 보류했다.
- AI 기반 개인 펫 시스템(AI 팅커벨 (Tech Impact))
  - 가족 구성원의 취향을 요약해 전달하는 가상 캐릭터
- 가족 간 용돈 관리
- 가계부 기능
  - 권한 모델 확장 예시로만 설계 문서에 포함
- 실시간 채팅 / 실시간 알림

## 🎨 Design Principles

- shadcn/ui 컴포넌트 사용
- 모바일 퍼스트, 앱 내 웹뷰 환경을 고려한 Web-in-App Experience
- 카드 기반 레이아웃으로 콘텐츠 구분 명확화
- 다크모드 기본 제공
- 모든 핵심 액션은 **3클릭 이내 완료**
